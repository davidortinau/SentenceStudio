# SentenceStudio Database Schema

Generated from `src/SentenceStudio.Shared/Data/ApplicationDbContext.cs` and the model files under `src/SentenceStudio.Shared/Models/`.

The same `ApplicationDbContext` is used on both **PostgreSQL** (server-side, via Aspire) and **SQLite** (mobile clients). ASP.NET Identity tables live on the server only (the SQLite build extends `DbContext`, the server build extends `IdentityDbContext<ApplicationUser>`).

> **Ownership column legend**
> - `UserProfileId` (string FK) — preferred ownership column. Used by mobile + most endpoints.
> - `UserId` (string FK) — legacy column on a handful of tables. Stores the same value as `UserProfileId`.
> - `Id == UserProfileId` — for `UserProfile`, the row's own Id IS the user.
> - **(none)** — table has no ownership column. ⚠️ Critical for sync isolation.

## 14. Cross-Provider Migration Notes

- Migrations live in two folders: `src/SentenceStudio.Shared/Migrations/Postgres/` and `src/SentenceStudio.Shared/Migrations/Sqlite/`. See `.squad/skills/ef-dual-provider-migrations/SKILL.md`.
