## Key Findings

- MainLayout.razor should NOT be an auth gate; Router (Routes.razor) is the official enforcement point
- Create MauiAuthenticationStateProvider wrapping IdentityAuthService to integrate with Blazor's auth framework
- AuthorizeRouteView replaces RouteView to render NotAuthorized inline instead of NavigateTo
- Remove boolean preferences (`app_is_authenticated`); use ClaimsPrincipal for auth state
- WebApp integration is low-risk (just add CascadingAuthenticationState to App.razor)
