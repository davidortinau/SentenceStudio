## Outcome

**Status:** COMPLETED ✅

### Summary
Fixed critical mobile authentication bypass vulnerability where the auth gate in `MainLayout.razor` only checked a boolean preference flag instead of validating actual token state. The preference persists across app restarts, allowing unauthorized access after a single login.

### Changes

- **MainLayout.razor:** Added async auth verification on layout initialization. Checks `IAuthService.IsSignedIn` and attempts silent refresh from SecureStorage/token cache. Clears preference and redirects to `/auth` if no valid auth state.
- **Auth.razor:** Profile selection now enforces server authentication via `LoginAsAsync` before setting preference. "Create Local User" redirects to `/auth/register` instead of bypassing auth.

### Testing

- 2 files changed
- Builds pass for iOS, WebApp, and MacCatalyst targets
- Auth flow validates against API server before any content access

### Key Decision

MainLayout.razor is now the single auth enforcement point. No local-only bypass allowed. Rules documented in `.squad/decisions/inbox/kaylee-mobile-auth-guard.md`.

## Impact

- **Wash (API):** No API changes required. JWT and DevAuthHandler flows unchanged.
- **Zoe (Lead):** Consistent with Phase 1 auth architecture decisions.
- **Jayne (QA):** Auth gate tests should verify: (a) fresh app launch redirects to /auth, (b) valid refresh token restores session, (c) expired tokens force re-login.
