## Key Findings

- Microsoft prescribes `AuthenticationStateProvider` as the core abstraction for auth in Blazor Hybrid
- Router uses `<AuthorizeRouteView>` (not plain `<RouteView>`) to enforce auth at routing level
- Inline component rendering in `<NotAuthorized>` slot replaces NavigateTo() redirects (which don't fire during OnInitializedAsync)
- Reactive state changes via `NotifyAuthenticationStateChanged()` enable proper lifecycle integration
- Pages use declarative `[Authorize]` attributes instead of manual layout gate logic

## Impact

- Eliminates NavigateTo() timing issues
- Follows Microsoft best practices (framework-designed pattern, not custom workarounds)
- Reduces MainLayout complexity (~60 lines removed)
- Enables standard Blazor auth components and role-based auth
