## Task

Fix #152 — Daily plan progress counter stays at 0/2 even after completing activities.

## Root Cause

`UpdatePlanItemProgressAsync` only updated `MinutesSpent`. It never checked if accumulated time met `EstimatedMinutes`, so `IsCompleted` stayed false and dashboard showed 0 completions.

## Solution Implemented

Added self-healing completion detection with defense-in-depth approach:

### 4 Changes in ProgressService.cs

1. **Time-based completion as truth source** (lines 611–630)
   - `EnrichPlanWithCompletionDataAsync` now computes `effectiveCompleted = IsCompleted || (MinutesSpent >= EstimatedMinutes)`
   - Resilient to races where `IsCompleted` flag wasn't committed yet

2. **Create-on-missing records** (lines 645–674)
   - `UpdatePlanItemProgressAsync` creates `DailyPlanCompletion` record when none exists
   - Includes duplicate protection for concurrent fire-and-forget saves

3. **No DB writes in read paths**
   - Self-healing is in-memory only for display
   - DB gets fixed on next `UpdatePlanItemProgressAsync` call

4. **Fallback plan initialization**
   - `GenerateFallbackPlanAsync` now calls `InitializePlanCompletionRecordsAsync`
   - Matches LLM plan initialization path
