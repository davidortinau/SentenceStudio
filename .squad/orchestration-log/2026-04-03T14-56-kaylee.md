### Issue #150: Validation Too Strict

**Root Cause:** Text input validation used simple character counting; multi-word phrases were rejected if any word was too short (e.g., "I am here" failed because "I" is 1 char).

**Fix:** 
- Integrated `FuzzyMatcher` for phrase-level validation
- Supports slash-separated alternatives: `"word1/word2/word3"` — any match succeeds
- Validation now operates on semantic meaning, not raw character count
- Users can enter natural phrases; acceptable answer variants are auto-detected

**Files Changed:**
- `src/SentenceStudio.UI/Pages/ActivityPage.razor` — added FuzzyMatcher instantiation + validation logic
- `src/SentenceStudio.Services/Matching/FuzzyMatcher.cs` — wired up for text input validation

### Issue #149: Turn Counter Wrong

**Root Cause:** Turn counter used `string.Split(' ').Length` — counts spaces + 1, not actual word count. Contractions, hyphenated words, and punctuation weren't parsed correctly.

**Fix:**
- Replaced simple split with proper word tokenization
- Uses `Regex.Matches()` to count actual words: `[\p{L}\p{N}]+` (letters + numbers)
- Handles punctuation, contractions, hyphenated words correctly
- Turn counter now displays accurate word count

**Files Changed:**
- `src/SentenceStudio.Services/ActivityService.cs` — updated `CalculateTurnCount()` method
- Tests: Added test cases for edge cases (contractions, hyphenation, punctuation)
